## المشكلة
الموظفين (مثل ياسمين) كانوا يلاقوا لخبطة لأن:
1. المهام بعد الأكشن كانت تظهر تاني في القائمة العادية → تم إصلاحه (المهمة تختفي وتنتقل لـ "تمت اليوم").
2. مفيش طريقة يشوفوا بيها مين عمل إيه ومتى لو مهمة اتنقلت أو اتعمل عليها أكتر من أكشن.

## النظام
### جدول `staff_task_action_log` (append-only)
- يسجّل كل INSERT/UPDATE على `staff_task_handling` تلقائياً عبر trigger `trg_log_staff_task_action`.
- مايتمسحش — حتى لو الـ handling record اتمسح، السجل التاريخي يفضل.
- **RLS**: كل الموظفين (admin + moderator) يقدروا يشوفوا كل السجل (شفافية كاملة).

### مكوّن `TaskActionHistoryDialog`
- زر **"سجل الإجراءات"** بأيقونة 🕐 في كل كارت مهمة بجوار "تفاصيل الدرجة".
- يفتح Dialog يعرض **كل الإجراءات** على المهمة دي (الأحدث أولاً) بتنسيق timeline:
  - أيقونة + بادج ملون لنوع الإجراء (call/whatsapp/done/note/outcome).
  - اسم الموظف + التاريخ والوقت.
  - الملاحظة (لو موجودة).
  - بادج "الأحدث" على أول إجراء.
- **Realtime**: لو موظف تاني عمل أكشن جديد والـ dialog مفتوح، السجل يتحدّث فوراً.

## الفايدة
- شفافية كاملة بين الفريق → "أحمد كلّمه الساعة 10، يسمين بعتته واتساب الساعة 11، محمد قفل المهمة 11:30".
- منع التكرار → الموظف يقدر يشوف لو زميل سبقه على مهمة قبل ما يكرر الاتصال.
- أرشيف للإدارة → ممكن مراجعة كل إجراء على أي مهمة في أي وقت.
